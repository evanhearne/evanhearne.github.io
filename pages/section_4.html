4. Evaluation of the Artefact

4.1 Reflection on meeting the Brief

For my system, I wanted it to be a seamless product, one in which the user wouldn't have to have too much technical knowhow to use. Unfortunately, due to time constraints I was not able to achieve this, and the microbit needed to be plugged into the computer for the advanced part of the task to be achieved. However, I was able to make a system that took inputs, and gave outputs. These outputs were able to be used to analyse for the user to make decisions in relation to social distancing guidelines. The product was also able to run autonomously once power was received, and the user didn't need to intervene unless taking data. In this case, the user had to manually stop the data collection, which gave them control. There was a client server model in which data was sent back to the microbit, which wrote this data to the computer. The microbits also communicated with each other to inform each other of signal strength. There was a Python program available to analyse data collected from the microbits, and the user could make an informed decision on their social habits with this data in mind.

4.2 Future Development of the Artefact

Of course, there are elements that could be improved with future iterations of the design cycle. For example, the microbits weren't entirely wireless - one had to be plugged into the computer to collect data. In a future iteration, I could work on this so that the system collects data completely wireless, or be able to store data on its own for analysis later. It isn't quite practical for people who are on the move all the time, as they would need to bring their laptop with them all the time. I would also like the Microbit devices to connect with any other microbit seamlessly, without constraints of the amount of microbits being used. I would like users to have had the ability to connect with various other microbit devices. Unfortunately, the microbits are limited to two devices at a time, as they were designed to be such. In future iterations, I would like to see seamless connections for ease of use. Also, the microbit itself is very limiting, as the sensors on each microbit aren't calibrated the same, which means there is room for error in terms of signal strength. This would be improved in future iterations, with devices calibrated the same to remove sources of error. The device could also be used to measure distance between two objects, provided the users new signal strength measurements, as it displays the signal strength on the screen while it runs.
